# 공문접수 관리 시스템 — GitHub + Replit 배포 가이드

> [입찰 시스템 인프라 가이드](https://github.com/plzkanu/bidding/blob/master/docs/인프라-아키텍처-가이드.md)와 동일한 **GitHub → Replit Republish** 방식을 사용합니다.

## 흐름 요약

1. **로컬(Cursor)** 에서 코드 수정 후 **GitHub** 로 푸시
2. **Replit Shell** 에서 GitHub 최신 코드로 동기화
3. **Replit** 에서 Republish 로 재배포

```
┌─────────────┐     git push      ┌──────────────┐
│ Cursor/로컬  │ ───────────────► │   GitHub     │
└──────┬──────┘                  └──────┬───────┘
       │ npm run dev (개발)              │ git fetch + build
       ▼                                 ▼
┌─────────────┐                  ┌──────────────┐
│ localhost   │                  │   Replit     │
│ FE :5173    │                  │ (Cloud Run)  │
│ BE :8000    │                  │ 단일 PORT    │
└─────────────┘                  └──────────────┘
```

| 계층 | 기술 | 프로덕션 |
|------|------|----------|
| 프론트엔드 | React + Vite | `frontend/dist` 정적 파일 |
| API | FastAPI | `/api/*` |
| DB | SQLite | `backend/official_doc.db` |
| 파일 | 로컬 디스크 | `backend/uploads`, `backend/assets` |
| 배포 | GitHub → Replit | CI/CD 없음 (수동 Republish) |

## 사전 준비 (최초 1회)

### 1. GitHub 저장소

| 항목 | 값 |
|------|-----|
| Remote | [https://github.com/plzkanu/official](https://github.com/plzkanu/official) |
| 기본 브랜치 | `master` |

최초 1회 푸시:

```bash
git init
git remote add origin https://github.com/plzkanu/official.git
git add .
git commit -m "Initial commit: 공문접수 관리 시스템"
git branch -M master
git push -u origin master
```

> `.env`, `*.db`, `uploads/` 파일은 `.gitignore`에 포함되어 Git에 올라가지 않습니다.

### 2. Replit 프로젝트 연결

1. [Replit](https://replit.com) → **Import from GitHub**
2. 저장소 선택 후 Import
3. `.replit` 설정이 자동 적용됩니다 (Build: `npm run build`, Run: `npm run start`)

### 3. Replit Secrets (필수)

**Tools → Secrets** 에 아래를 설정합니다.

| Secret | 용도 |
|--------|------|
| `SECRET_KEY` | JWT 서명 키 (`openssl rand -base64 32` 등으로 생성) |

### 4. Replit Secrets (선택 — SMTP)

| Secret | 용도 |
|--------|------|
| `SMTP_HOST` | SMTP 서버 |
| `SMTP_PORT` | SMTP 포트 (기본 587) |
| `SMTP_USER` | SMTP 계정 |
| `SMTP_PASSWORD` | SMTP 비밀번호 |
| `SMTP_FROM` | 발신 주소 |

SMTP 미설정 시 담당부서 알림은 서버 콘솔에 출력됩니다.

## 로컬 개발

### 방식 A — 프론트/백엔드 분리 (권장)

**터미널 1 — Backend**

```bash
cd backend
python -m venv venv
venv\Scripts\activate          # Windows
pip install -r requirements.txt
copy .env.example .env
uvicorn app.main:app --reload --port 8000
```

**터미널 2 — Frontend**

```bash
cd frontend
npm install
npm run dev
```

브라우저: http://localhost:5173

### 방식 B — 프로덕션과 동일하게 (통합 서버)

```bash
npm install
npm run build
set SECRET_KEY=local-test-secret   # Windows
set STATIC_DIR=../frontend/dist
npm run start
```

브라우저: http://localhost:8000

## 배포 절차 (매번)

### 1단계: 로컬에서 GitHub 푸시

```bash
git add .
git commit -m "변경 내용 설명"
git push origin master
```

### 2단계: Replit Shell 동기화 + 빌드

Replit **Tools → Shell** 에서:

```bash
git fetch origin master && git reset --hard origin/master && npm install && npm run build
```

> 브랜치가 `main`이면 `master`를 `main`으로 바꿉니다.

빌드 로그에 아래가 보이면 정상입니다.

```
[install-python-deps] venv 생성: .venv
[install-python-deps] pip install backend/requirements.txt
[build] frontend vite build
```

### 3단계: Republish

1. Replit 오른쪽 상단 **Deploy → Republish**
2. 배포 URL 접속
3. `/login` → 로그인 → 대시보드 확인

## Replit 실행 설정 (`.replit`)

| 항목 | 값 |
|------|-----|
| Build | `npm run build` |
| Run | `npm run start` → `scripts/start-production.mjs` |
| Deployment target | Cloud Run |
| Modules | nodejs-22, python3 |
| Port | Replit `PORT` → uvicorn `0.0.0.0` |

`npm run start`는 FastAPI(uvicorn)를 `backend/`에서 실행하며, `STATIC_DIR=../frontend/dist` 로 프론트엔드를 함께 서빙합니다.

## 반영이 안 될 때

| 증상 | 조치 |
|------|------|
| 예전 코드가 배포됨 | Shell에서 `git fetch && git reset --hard origin/master` 재실행 |
| 빌드 실패 | Shell에서 `npm run build` 로 오류 확인 |
| 502 / 서버 기동 실패 | Secrets에 `SECRET_KEY` 설정 확인 |
| `프론트엔드 빌드 결과를 찾을 수 없습니다` | `npm run build` 재실행 후 Republish |
| 로그인 후 401 | Replit Republish 후 브라우저 캐시 삭제 |
| 업로드 파일 유실 | Replit 재배포 시 SQLite·uploads는 Replit 디스크에 유지됨. Repl 삭제 시 데이터 소실 주의 |
